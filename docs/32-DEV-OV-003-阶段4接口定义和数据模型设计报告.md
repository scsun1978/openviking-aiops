## 1. 模块间接口定义

### 1.1 边缘计算模块 → AI 智能运维模块

**接口名称**：边缘资源优化接口

**协议**：HTTP + RESTful API

**端点**：
```
POST /api/edge/aiops/optimize
```

**请求**：
```json
{
  "edge_id": "edge-001",
  "optimization_type": "memory",  # memory, cpu, disk
  "target": {
    "usage_percent": 70,
    "performance": "latency"
  },
  "constraints": {
    "max_time": 300,  # seconds
    "max_restart": 1
  }
}
```

**响应**：
```json
{
  "optimization_id": "opt-001",
  "status": "pending",
  "estimated_time": 300,
  "created_at": "2026-03-23T12:00:00Z",
  "updated_at": "2026-03-23T12:00:00Z"
}
```

**错误码**：
- `400`：请求参数错误
- `404`：边缘设备不存在
- `409`：优化任务冲突
- `500`：服务器内部错误

**接口说明**：
- 调用 AI 智能运维模块进行边缘资源优化
- AI 智能运维模块分析边缘设备资源使用情况
- 返回优化建议（如内存优化、CPU 优化）
- 可以选择是否执行自动优化

## 3. 数据模型设计

### 3.1 监控数据模型

**表名**：`metrics`

**字段**：
| 字段名 | 类型 | 说明 |
|--------|------|------|
| `metric_id` | String | 指标唯一标识（主键） |
| `metric_name` | String | 指标名称（如：cpu.usage、memory.usage） |
| `metric_type` | String | 指标类型（gauge、counter、histogram） |
| `metric_value` | Float | 指标值 |
| `metric_unit` | String | 指标单位（如：%、ms、Mbps） |
| `edge_id` | String | 边缘设备 ID（外键） |
| `server_id` | String | 服务器 ID（外键） |
| `timestamp` | DateTime | 指标时间戳 |
| `created_at` | DateTime | 创建时间 |
| `updated_at` | DateTime | 更新时间 |

**索引**：
- `metric_id`：主键索引
- `metric_name`：普通索引
- `edge_id`：普通索引
- `server_id`：普通索引
- `timestamp`：普通索引

## 4. 数据流设计

### 4.1 监控数据流

```
边缘设备 → 监控代理 → Prometheus → 数据库 → 监控告警模块 → VikingBot → OpenViking
    ↓           ↓           ↓          ↓              ↓          ↓
  原始数据    预处理      存储      查询          告警        知识库
```

### 4.2 日志数据流

```
边缘设备 → 日志收集 → Loki → 数据库 → AI 智能运维模块 → VikingBot → OpenViking
    ↓           ↓         ↓          ↓               ↓          ↓
  原始日志    解析        存储      查询           分析        知识库
```

### 4.3 告警数据流

```
监控模块 → 监控告警模块 → 告警引擎 → 告警通知 → 用户 / 自动修复
    ↓              ↓              ↓           ↓             ↓
  指标        告警          规则        通知        操作
```

### 4.4 AI 智能运维数据流

```
监控数据 / 日志 → AI 智能运维模块 → VikingBot → LLM API → OpenViking → 知识库
      ↓               ↓               ↓          ↓          ↓
    原始数据        分析           推理        存储      检索
```

### 4.5 运维知识数据流

```
运维案例 / 日志 / 告警 → AI 智能运维模块 → VikingBot → LLM API → OpenViking
      ↓                  ↓               ↓          ↓
    原始数据            提取           推理        存储
```
