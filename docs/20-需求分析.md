## 1. 项目概述

### 1.1 项目背景

基于已部署的 OpenViking 项目（上下文数据库）和 OpenViking Bot（AI 智能体），运行在边缘计算设备上（172.16.100.101:22），开发一个基于边缘计算设备的 AI 运维系统，实现整个网络的智能化运维。

### 1.2 项目目标

创建一个完整的 AI 运维系统，包括：
1. **边缘计算模块**：Docker 容器化部署、资源管理
2. **网络运维模块**：网络状态监控、设备健康度监控、告警规则引擎
3. **服务器运维模块**：服务器资源监控、系统状态监控、服务管理、日志管理
4. **AI 智能运维模块**：日志智能分析、异常智能检测、故障智能诊断、智能问答
5. **监控告警模块**：设备健康度监控、网络性能监控、LLM 智能运维性能监控
6. **运维知识管理模块**：OpenViking 知识库、运维知识检索、运维经验学习

### 1.3 核心原则

**研发层原则**：
1. **OpenClaw 负责总控与任务编排** - 任务下发、进程管理、会话协调
2. **Claude Code 负责代码修改与执行** - 代码开发、测试、调试
3. **Obsidian 负责项目知识沉淀** - 需求分析、架构设计、最佳实践记录
4. **Git / CI 负责质量门禁** - 代码审查、自动化测试、质量门禁

**运维层原则**：
1. **OpenViking 不参与研发主流程** - 仅作为运维环境的上下文数据库
2. **OpenViking Bot 负责 LLM 智能运维推理** - 运维决策、故障诊断、自动修复建议
3. **运维产品在运行时与 LLM 交互** - 通过 Bot 接口进行智能运维推理
4. **运维经验沉淀在 OpenViking** - 不回流到研发层，只在运维环境内循环

### 1.4 关键约束

**边缘计算限制**：
- CPU：Intel Xeon Silver 4210R @ 2.40GHz（核心数待确认）
- 内存：31Gi（可用 28Gi，使用率 < 80%）
- 存储：250G（可用 158G，使用率 < 90%）
- 网络：1Gbps（延迟 < 10ms）

**实时性要求**：
- LLM 智能运维决策延迟：< 100ms
- 处理吞吐量：> 100 请求/秒
- 系统可用性：> 99.9%
- 告警响应时间：< 1 分钟

**安全性要求**：
- 数据隐私：数据不离开边缘设备
- 加密传输：所有网络通信加密
- 访问控制：严格的 RBAC

**可扩展性要求**：
- 水平扩展：支持添加更多边缘设备
- 模型更新：通过 LLM API（无需本地训练）
- 功能扩展：模块化设计，易于扩展

## 4. 技术选型

### 4.1 技术栈总览

| 组件 | 技术选型 | 版本 | 说明 |
|------|----------|------|------|
| **容器化** | Docker | 24.0+ | 容器化部署 |
| **编排** | Docker Compose | 2.20+ | 服务编排 |
| **LLM 集成** | VikingBot 集成 | latest | LLM API 集成 |
| **LLM 模型** | 火山引擎豆包 | doubao-seed-2-0-pro-260215 | 智能运维 |
| **监控** | Prometheus + Grafana | latest | 监控和可视化 |
| **告警** | AlertManager | latest | 告警管理 |
| **日志** | Loki | latest | 日志收集和分析 |

### 4.2 技术选型理由

#### Docker + Docker Compose（推荐）

**优势**：
- 简单易用，学习成本低
- 轻量级，资源占用低
- 已部署 OpenViking + VikingBot 使用 Docker，技术栈一致
- 符合边缘计算设备限制（31Gi 内存）

**劣势**：
- 缺少自动编排能力
- 缺少自愈能力
- 扩展性有限

#### VikingBot 集成（推荐）

**优势**：
- 已集成火山引擎豆包 LLM
- 开箱即用，无需额外配置
- OpenViking 集成，知识管理方便
- 轻量级，资源占用低
- 已部署，无需额外部署

**劣势**：
- 仅限 VikingBot 内使用
- 不支持外部应用调用

#### Prometheus + Grafana（推荐）

**优势**：
- 已部署，无需额外部署
- 生态成熟，功能强大
- 支持多种数据源
- 丰富的 Grafana 面板
- 强大的告警规则引擎

**劣势**：
- 学习成本高
- 需要维护
