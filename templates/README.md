# 模板使用指南

> 本指南说明如何使用项目中的各类模板，实现 OpenClaw 任务下发、Claude Code 执行、ADR 记录三者的统一流程。

---

## 模板概览

| 模板 | 文件路径 | 用途 | 使用者 |
|------|----------|------|--------|
| 任务模板 | `templates/任务模板.md` | 创建标准任务文件 | OpenClaw |
| ADR 模板 | `templates/ADR-模板.md` | 记录架构决策 | Claude Code / 人工 |
| OpenClaw 下发模板 | `templates/openclaw-任务下发模板.md` | 下发任务给 Claude Code | OpenClaw |
| Claude Code 执行模板 | `templates/claude-code-执行模板.md` | 执行任务的标准流程 | Claude Code |

---

## 完整工作流程

```
┌─────────────┐         ┌──────────────┐         ┌─────────────┐
│  OpenClaw   │────────>│   任务文件   │<────────│  Claude Code│
│  任务下发   │         │ tasks/DEV-XXX│         │  执行并回写 │
└─────────────┘         └──────────────┘         └─────────────┘
       │                                                    │
       │                                                    │
       v                                                    v
┌─────────────┐         ┌──────────────┐         ┌─────────────┐
│   使用模板   │         │   ADR 记录   │<────────│  需要决策时  │
│   下发任务   │         │07-决策记录   │         │  创建 ADR   │
└─────────────┘         └──────────────┘         └─────────────┘
```

## 步骤 1：创建任务文件

### 使用场景

OpenClaw 需要下发新任务时

### 操作步骤

1. **复制任务模板**
   ```bash
   # 复制模板到新任务文件
   cp templates/任务模板.md tasks/DEV-XXX.md
   ```

2. **填写任务信息**
   ```markdown
   ## 基本信息
   - 任务编号：DEV-XXX
   - 任务名称：[简短描述]
   - 状态：TODO
   - 优先级：高/中/低
   - 创建日期：YYYY-MM-DD

   ## 任务目标
   [1-2 句话描述目标]

   ## 范围
   - [ ] 范围项 1
   - [ ] 范围项 2

   ## 非目标
   - 不做 xxx
   - 不涉及 xxx

   ## 验收标准
   1. [ ] 可测试条件 1
   2. [ ] 可测试条件 2
   ```

3. **状态标记**
   - 初始状态为 `TODO`
   - Claude Code 开始执行时更新为 `DOING`
   - 完成时更新为 `DONE`

## 步骤 2：下发任务给 Claude Code

### 方式 A：完整模板下发

适用于复杂任务，需要详细说明

```
请阅读 tasks/DEV-XXX.md 了解任务详情。

请按照 templates/claude-code-执行模板.md 的标准流程执行：

1. 先进行只读分析，理解代码结构
2. 给出实施方案，等待确认
3. 确认后开始编码
4. 执行测试验证
5. 输出完整总结和任务纪要

特别注意：
- 风险边界：[列出禁止的操作]
- 验收标准：[强调关键验收条件]
```

### 方式 B：快速任务下发

适用于简单任务

```
项目：[项目名]
任务编号：DEV-XXX
任务文件：tasks/DEV-XXX.md

目标：[一句话描述目标]
验收标准：
1. [标准 1]
2. [标准 2]

请按标准流程执行。
```

## 步骤 3：Claude Code 执行任务

### 执行前准备

```python
# 1. 读取任务文件
Read tasks/DEV-XXX.md

# 2. 读取执行模板
Read templates/claude-code-执行模板.md

# 3. 理解任务需求和验收标准
```

### 按流程执行

```yaml
# 阶段 1：只读分析
- 使用 Glob、Read、Grep 理解代码
- 识别涉及模块和文件
- 评估风险点
- 输出分析结果

# 阶段 2：方案确认
- 设计实施方案
- 列出修改计划
- 等待确认

# 阶段 3：编码实现
- 按方案实施
- 遵循代码规范
- 检查代码质量

# 阶段 4：测试验证
- 执行测试命令
- 记录测试结果
- 验证验收标准

# 阶段 5：输出总结
- 生成文件清单
- 生成 PR 摘要
- 生成回归建议
- 生成任务纪要
```

### 回写任务文件

```python
# 更新任务文件的执行记录部分
Edit tasks/DEV-XXX.md

# 添加或更新执行记录
### 第 N 次执行
- 时间：YYYY-MM-DD HH:MM
- 阶段：[阶段名称]
- 输出：
  - ✅ [完成的输出项]
- 问题：
  - ⏳ [遗留问题]

# 如果任务完成，更新状态和完成说明
## 最终结果
- [x] 已完成

## 完成时间
YYYY-MM-DD HH:MM

## 完成说明
1. ✅ [完成项 1]
2. ✅ [完成项 2]
```

## 步骤 4：创建 ADR（如需要）

### 使用场景

任务过程中有重要技术决策需要记录

### 操作步骤

1. **判断是否需要 ADR**
   - 技术选型决策
   - 架构设计决策
   - 影响范围较大的决策
   - 需要长期维护的决策

2. **创建 ADR**
   ```bash
   # 在 07-决策记录-ADR.md 中添加新 ADR
   # 使用 templates/ADR-模板.md 作为格式参考
   ```

3. **ADR 格式**
   ```markdown
   ## ADR-XXX：[决策标题]

   ### 元信息
   - 编号：ADR-XXX
   - 状态：已接受
   - 日期：YYYY-MM-DD
   - 相关任务：DEV-XXX

   ### 背景
   [为什么需要这个决策]

   ### 决策
   [我们决定做什么]

   ### 原因
   1. 原因 1
   2. 原因 2

   ### 影响
   - 积极影响
   - 消极影响/风险
   - 缓解措施
   ```

4. **关联到任务文件**
   ```markdown
   ## 决策记录（ADR）
   - ADR-XXX：[决策标题]
   ```

## 模板关系图

```
templates/任务模板.md
        │
        ├─> 复制到 tasks/DEV-XXX.md
        │        │
        │        └─> Claude Code 读取并执行
        │                 │
        │                 └─> 回写执行记录到 tasks/DEV-XXX.md
        │
templates/openclaw-任务下发模板.md
        │
        └─> OpenClaw 使用此格式下发任务
                 │
                 └─> 引用 tasks/DEV-XXX.md

templates/claude-code-执行模板.md
        │
        └─> Claude Code 按此流程执行
                 │
                 ├─> 输出结果
                 └─> 回写到 tasks/DEV-XXX.md

templates/ADR-模板.md
        │
        └─> 需要决策时使用
                 │
                 └─> 添加到 07-决策记录-ADR.md
                          │
                          └─> 在任务文件中引用
```

## 复制到新项目

### 模板文件

```bash
# 复制整个 templates 目录
cp -r templates/ /path/to/new-project/

# 或者复制单个模板
cp templates/任务模板.md /path/to/new-project/templates/
cp templates/ADR-模板.md /path/to/new-project/templates/
cp templates/openclaw-任务下发模板.md /path/to/new-project/templates/
cp templates/claude-code-执行模板.md /path/to/new-project/templates/
```

### 目录结构

新项目应包含：

```
new-project/
├── templates/
│   ├── 任务模板.md
│   ├── ADR-模板.md
│   ├── openclaw-任务下发模板.md
│   ├── claude-code-执行模板.md
│   └── README.md
├── tasks/
│   ├── DEV-001.md
│   ├── DEV-002.md
│   └── _index.md
└── 07-决策记录-ADR.md
```

## 验收标准检查

### 标准 1：至少一个真实任务按模板完成

检查项：
- [ ] 任务文件使用 `templates/任务模板.md` 创建
- [ ] Claude Code 按 `templates/claude-code-执行模板.md` 执行
- [ ] 执行记录完整回写到任务文件
- [ ] 验收标准全部达成

### 标准 2：模板可复制到新项目

检查项：
- [ ] 模板文件独立完整
- [ ] 不包含项目特定内容
- [ ] 有清晰的使用说明
- [ ] 可直接复制使用

### 标准 3：模板可支持任务纪要回写

检查项：
- [ ] Claude Code 执行模板包含纪要输出格式
- [ ] 任务文件包含执行记录部分
- [ ] 有明确的回写示例
- [ ] 验证过实际回写流程

## 常见问题

### Q1：任务状态如何流转？

```
TODO → DOING → REVIEW → DONE
          ↓
       BLOCKED
```

- **TODO**：待开始
- **DOING**：执行中
- **REVIEW**：待审核
- **DONE**：已完成
- **BLOCKED**：被阻塞

### Q2：什么时候创建 ADR？

- 技术选型决策（框架、库、工具）
- 架构设计决策（模块划分、接口定义）
- 影响范围较大的决策
- 需要长期维护和追溯的决策

### Q3：执行记录如何组织？

每次执行添加一个「第 N 次执行」部分：
- 时间
- 阶段
- 输出（使用 ✅ 标记）
- 问题（使用 ⏳ 标记遗留，⚠️ 标记障碍）
- 下一步

### Q4：如何确保模板一致性？

1. 使用统一的模板文件
2. 定期检查任务文件格式
3. 通过 Code Review 验证
4. 使用此指南作为参考
